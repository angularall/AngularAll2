Case Study:
 
DealsHub — a simplified deal hunting experience!
 
Develop a pseudo front end application which would let the users list online
Deals.
You are supposed to implement frontend of deals indexing service. Data will be rendered in JSON format, parse it and do listing using pagination.
 
[Minimum Requirement] (Entry level)
 
— Use of Web API to fetch deal details.
 
* API Parameters:
deals[] —[ name, actual_price, discount, rating, provider, link, image ].
— Visually interactive responsive design listing all the deals.
— Submit Screenshot, Source code & Instructions.
 
[Plus Point] (Intermediate)
 
— Display a Final Price based on actual price and discount parameter.
 
* final_price = actual_price - discounted amount.
 
— Display deal details including deal name, image, actual_price, final_price, discount, rating and link.
— Link to provider’s portal. (Open in new window | target="_blank")
 
[Extra Work] (Experienced / wizard / if time permits)
 
— Display and keep track of number of total Likes.
— A feature to Like deal and maintain total likes. (HTML5 Localstorage)
— Custom elegant design, fonts and icons to make app more user-friendly.
— You may add portfolio activity comprising awesome work you have done on web application.
— Use your imagination and add features which would make things easier for end users.
 
[Guide]
 
— Deal details API: http://starlord.hackerearth.com/cognizantinternal/dealshub
 
— Ideal Frontend Stack:
*JS Framework: Javascript, AngularJS, jQuery.
* Bootstrap, Typography, Subtle pattern, Custom Icons.
 
— PS: Application can be built with use of client-side scripting and need not require any backend.
— HTML5 Local Storage: http://html5demos.com/storage
 
— For fonts, icons and subtle texture:
- https://github.com/bperin/FontAwesomeAndroid
- http://glyphsearch.com/
- http://subtlepatterns.com/thumbnail-view/
- https://www.google.com/fonts
- http://www.flatuicolorpicker.com/green
- http://getbootstrap.com/components/
 
API Specs
 
API Status: https://hackerearth.0x10.info/api/uptime
 
Supports GET/POST/PUT requests on HTTP as well as HTTPS.
Output Response: JSON, CSV or XML
 
Example urls:
* https://example.com/api/deals?type=json&query=list_deals
* https://example.com/api/deals?type=json&query=api_hits
 
HTTP Header:
- access-control-allow-headers:Origin, X-Requested-With, Content-Type, Accept
- access-control-allow-methods:GET, POST, PUT
- access-control-allow-origin:*
- server:cloudflare-nginx
 
[Wireframe MVP / Reference Structure]
 
  • Mockup image with instructions
  • App Page I — Home page comprising indexing of deals.




**This mockup is just for reference, we expect a way much more creative design**
 
Structure / Data Parameters:
 
deals[] —[ name, actual_price, discount, rating, provider, link, image ].
* name AS text (deal name)
* actual_price AS number (actual price)
* discount AS percentage (available discount over actual price)
* rating AS float (rating of deal)
* provider AS text (deal provider)
* link AS text (link of portal provider)
* image AS text (image link)
